Enterprise Azure ANF

Sovereign agentic factory operations on Azure NetApp Files.

NemoClaw is the project-owned overlay for an independent technical architecture effort that combines Field Agent orchestration, Vault Agent secret handling, NVIDIA NIM inference, Nemotron reasoning, PostgreSQL state, and pgvector retrieval into a single public reference stack.

Architecture at a glance

  • Azure NetApp Files anchors the durable workspace and data paths.
  • NVIDIA NIM serves model access while Nemotron provides the reasoning layer.
  • PostgreSQL and pgvector preserve state, retrieval, and grounded memory.

Technology stack

The stack is intentionally narrow and explainable.

The public surface keeps the core platform visible: Azure NetApp Files for resilient storage, NVIDIA NIM for model serving, Nemotron for the reasoning model, PostgreSQL for system state, and pgvector for semantic retrieval.

Azure NetApp Files

High-performance shared storage for durable project state, artifact exchange, and sandbox-backed workflows.

NVIDIA NIM

Inference endpoints that expose the model layer through a controlled, production-shaped interface.

Nemotron

The reasoning model used to keep the architecture grounded in structured, explainable outputs.

PostgreSQL

The system of record for metadata, operational state, and delivery tracking.

pgvector

Vector search for retrieval-augmented responses, semantic lookup, and reusable context.

Field Agent / Vault Agent

Field Agent handles execution; Vault Agent protects the trust boundary.

Field Agent coordinates the operational workflow across the sandbox, while Vault Agent keeps secrets isolated, scoped, and auditable. The pairing is meant to be legible to reviewers rather than magical.

Field Agent

Task-focused orchestration for data gathering, content assembly, and delivery checks within the project-owned site workflow.

Vault Agent

Secret access broker that limits blast radius, preserves traceability, and keeps the assistant boundary explicit.

ANF value

Azure NetApp Files is the storage and collaboration backbone.

The ANF layer gives the factory a shared substrate for low-latency files, stable handoffs, and repeatable artifact publishing without making the site depend on fragile runtime state.

Durable shared state

Supports predictable read/write access for the assistant workflow and generated artifacts.

Operational separation

Keeps platform data, reasoning state, and published outputs cleanly separated.

Project-owned publishing

Makes the artifact store visible from the public site without introducing nested app routes.
