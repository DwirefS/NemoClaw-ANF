Security

Field-agent and vault-agent trust separation matters.

The architecture does not treat every agent as equally trusted. Field agents operate in a lower-trust zone with sanitized inputs, while vault agents handle privileged transitions, policy mediation, and the more sensitive retrieval surfaces.

Trust model

  • Field-agent / vault-agent separation defines the public trust boundary.
  • ACL-aware retrieval is the correct access model for sensitive corpora.
  • Worker execution remains separate from the high-trust data plane.
  • Policy, redaction, and audit stay visible in the architecture narrative.

Boundaries

Security is built around explicit handoffs, not blanket trust.

The pages describe how data moves between trust zones so review can focus on the actual transition points instead of assuming the assistant can read everything everywhere.

Field agents

Operate with constrained data and narrow scope so they can gather context without broad access.

Vault agents

Handle privileged retrieval and secret-aware operations under policy and audit control.

Retrieval policy

ACL-aware retrieval keeps access decisions attached to the requested corpus, not to a generic model prompt.

Worker isolation

NemoClaw/OpenShell workers remain separate so the execution tier does not inherit the data plane trust level.

Controls

The model makes policy enforcement visible.

The architecture uses mediation, masking, and bounded retrieval to keep the public story aligned with the actual security posture.

Policy mediation

Requests cross from low trust to high trust only after the appropriate checks and gates run.

Audit and redaction

Sensitive data handling is framed in terms reviewers can inspect rather than hidden behind a single assistant surface.
