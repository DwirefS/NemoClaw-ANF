## Use This Skill When

- you need to know which research source is canonical
- two architecture docs disagree
- you need provenance for a statement in the incubator
- you need to tell current-state facts from target-state proposals

## Workflow

- Check the canonical source map first.
- Use the manifest for file lineage and source classification.
- Use the contradiction graph when a claim appears to conflict with repo reality.
- Open raw PDFs, DOCX, or JSON only when the incubator corpus does not answer the question.

## Common Mistakes

- Treating a rendered PDF as more authoritative than its matching Markdown source.
- Treating the whitepaper as proof of current upstream NemoClaw behavior.
- Ignoring the DOCX draft when runtime maturity is in question.
